Offentlig retning

Mer enn en matolje. Ditt daglige, flytende kosttilskudd.

Previewen bruker kundens kildetekst som offentlig startpunkt. Dokumentasjon, analysebevis og trygg handel er bygget inn i opplevelsen, men brukes ikke som salgspitchen på forsiden.

Start her

Anbefalt rekkefølge

00 Preview v1 status

Start her for kort status: hva kan deles nå, og hva som fortsatt blokkerer produksjon.

 01 Forside

Sjekk merkevarefølelse, hero, produktfortelling, kjøpsflyt, FAQ, support og betalings-/fraktseksjon.

 02 Produktside

Se størrelsesstatus, kjøpsvalg, analysebevis-status, daglig bruk, sticky mobilkjøp og FAQ.

 03 Kildestatus

Se hva som er levert tekst, kravspesifikasjon, placeholder, fjernet mock-data og produksjonsrisiko.

 04 Trygg copy

Sammenlign en mer produksjonssikker copy-retning uten sykdoms- eller medisinpåstander.

 05 Produksjonsbrief

Se hvordan previewen bør gjøres om til Shopify-produkter, betaling, frakt, SEO, consent og support.

 06 Shopify datakontrakt

Se hvilke produkt-, variant-, metafield-, COA-, schema- og supportfelter som må være autoritative.

 06B Shopify theme handoff

Se hvordan theme ZIP-en skal lastes opp som draft, testes og holdes upublisert til launch-gates er grønt.

 07 Kildegrunnlag

Se hvilke 2026-kilder som styrer Shopify, betaling, frakt, SEO, tilgjengelighet og AI-retning.

 08 Checkout readiness

Se Vipps, kort, Posten/Bring, SSL, testordre, fast levering og opplæring før live.

 09 SEO/AEO readiness

Se indeksering, Merchant Center, schema og launch-gates før noindex fjernes.

 10 Påstandsgjennomgang

Se røde claims, dokumentasjonskrav og tryggere live-retning før copy og support låses.

 11 Support KB review

Se kunnskapsbase, intents, svargrenser og testmatrise før live FAQ-agent eller chatbot.

 12 Supportagent blueprint

Se produksjonsarkitektur for OpenAI, Shopify, personvern, guardrails og rollout før live agent.

 13 Supportagent testbenk

Test spørsmål, medisinsk handoff, produksjonsmodus, kildestatus og eksportert testrapport.

 14 Accessibility readiness

Se WCAG 2.2, keyboard, fokus, mobil, video, PDF/COA og checkout-test før live.

 15 Produsentvideo handoff

Se hvilke filer, rettigheter, teksting, transkript og metadata som må inn før videoen publiseres.

 16 Visuell retning

Se hvordan realfoto, produsentvideo og GPT Image 2-assets kan brukes uten falske produktbevis.

 17 Juridisk og policy

Se hvilke vilkår, selgerdata, personvern, cookies, frakt/retur og produktmerking som må være klare.

 18 Performance readiness

Se Core Web Vitals, mediebudsjett, Shopify-app-risiko og tester som må kjøres før live.

 19 Opplæring og drift

Se hvilke Shopify-oppgaver, roller, øvelser og handoff-rutiner kunden må mestre før overtakelse.

 20 Produksjonsintake

Samle produktdata, COA, media, claims, kontoer og betalings-/fraktavklaringer.

 21 Produksjonsforespørsel

Send én tydelig e-post med filstruktur, datakrav og mottakskriterier før Shopify-bygg.

 22 Lanseringssjekkliste

Se faseplan, operativ Shopify-sjekkliste, testordre og go/no-go før live.

 23 Kravdekning

Se hvordan opprinnelig brief er dekket i previewen og hvilke krav som fortsatt krever kundedata.

 24 Beslutningslogg

Marker hva som er godkjent, hva som må endres, og hva som fortsatt mangler før Shopify-produksjon.

 25 Kildeverifisering

Kontroller offisielle kilder og produksjonsantakelser før Shopify, SEO, betaling, frakt og support låses.

Mangler før produksjon

Dette må kunden levere

Produkt og innhold

  • Endelige produktnavn og hvilke to produkter som skal lanseres.
  • To størrelser per produkt, priser, SKU, vekt og lagerstatus.
  • Produktbilder, emballasjefiler og endelige logoeksporter.
  • Ingredienser, anbefalt daglig dose, porsjoner og advarsler.
  • COA/analysebevis, batch-ID, lab, metode, dato og sertifikat-PDF.
  • Produsentvideo fra Hellas med posterbilde, teksting og transkript.

Teknisk og juridisk

  • Juridisk godkjente produktpåstander eller kontaktperson for claim-review.
  • Shopify-konto eller utviklingsbutikk.
  • Shopify Payments eller valgt kortleverandør.
  • Vipps/MobilePay-oppsett og avklaring av abonnement/fast levering.
  • Posten/Bring-avtale og fraktvalg.
  • Cookie-/personvernpolicy, kjøpsvilkår, returvilkår og GA4/Search Console.

Produksjonsrisiko

Ikke lås dette uten avklaring

Helsepåstander

Claims om sykdom, blodtrykk, biomarkører, medisiner eller behandling må juridisk godkjennes før live.

Analysebevis

Previewen viser retningen, men reelle COA-data må inn før analysebevis kan være produksjonsinnhold.

Fast levering

Abonnement må testes med valgt Shopify subscription-app og betalingsoppsett før det loves.

Vipps/MobilePay

Vipps-ruten må verifiseres i faktisk Shopify checkout, spesielt hvis abonnement fortsatt er ønsket.

Genererte visuals

Previewbilder kan brukes til retning, men produksjon bør bruke ekte produktfoto, video og dokumenter.

Neste møte

Foreslått agenda

  1. Gå gjennom forside og produktside på desktop og mobil.
  2. Gå gjennom kildestatus og bekreft at ingen fiktive produktdata skal inn igjen.
  3. Gå gjennom claim review og marker hva som ikke skal brukes live.
  4. Fyll ut produksjonsintake med det som allerede er klart.
  5. Bruk lanseringssjekklisten for å prioritere Shopify, betaling, frakt, QA og opplæring.
  6. Avklar hvilke produktdata som kan låses nå.
  7. Avklar claims som må sendes til juridisk vurdering.
  8. Bestem om fast levering skal være første lansering eller fase 2.
  9. Planlegg Shopify-oppsett, Vipps/MobilePay, Posten/Bring og produksjons-QA.
