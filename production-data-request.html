Formål

Målet er å få ja, nei eller mangler på hvert produksjonskritiske punkt.

Previewen kan vurderes nå, men Shopify, checkout, frakt, SEO/schema og support bør ikke låses før disse dataene er levert eller eksplisitt markert som manglende.

Mottakskriterier

Produkter

2 produkter og 2 størrelser per produkt har navn, pris, SKU/GTIN hvis tilgjengelig, vekt og lagerstatus.

Analysebevis

COA er en faktisk fil eller stabil lenke, og batch/lab/metode/dato/verdier kan spores til dokumentet.

Claims

Helse- og effektpåstander er enten godkjent for bruk eller markert som ikke klare for live copy.

Media

Logo, packshots og video kan brukes kommersielt, og video har poster, captions, transkript og rettigheter.

Checkout

Shopify, Vipps/MobilePay og Posten/Bring har konkret eier, status og testplan.

Drift

Ansvar for produkter, COA, ordre, refund, frakt, support, analytics og teknisk eskalering er navngitt.

Neste handling

Bruk forespørselen som første produksjonsrunde

01 Send preview-hub

Bruk hub-lenken slik at mottaker ser status, reviewrekkefølge og produksjonsforbehold.

 02 Fyll intake

Samle data i skjemaet og eksporter `.md` eller `.json` for videre arbeid.

 03 Logg beslutninger

Marker godkjent retning, endringsønsker og manglende data etter review.

 04 Bygg Shopify

Først når dataene og kontoene er klare, bør theme, checkout, frakt og schema låses.
