Supportagent produksjonsblueprint

Bygg agenten som et kontrollert supportprodukt, ikke som en fri chatbot.

Dette er produksjonsoppskriften for OliveX-agenten: godkjent kunnskapsbase, OpenAI Responses/File Search, server-side nøkler, Shopify-grenser, personvern, guardrails, test og gradvis rollout.

Arbeidsregel

Agenten kan først gå live når den har færre frihetsgrader enn FAQ-en.

Den skal svare fra godkjente kilder, vise når den mangler grunnlag, eskalere helse-/medisinsaker og aldri hente produktfakta, leveringstid, claims eller rabattvilkår fra egen fantasi.

Arkitektur

Anbefalt produksjonsløype

1. Shopify widget

En lett chat/FAQ-widget i temaet. Ingen API-nøkler i nettleseren. Norsk språk, foreslåtte spørsmål og tydelig support-handoff.

2. Server endpoint

Shopify app proxy, serverless endpoint eller backend som holder OpenAI-nøkkel, rate limiting, logging og abuse-beskyttelse.

3. Retrieval

OpenAI Responses API med `file_search` mot godkjent vector store. Returner korte svar med kildeetikett.

4. Guardrails

Pre-filter for medisinsk/claim-sensitivt språk, systeminstruks, draft-kildeblokkering og eskalering ved usikkerhet.

5. Shopify data

Produkt-, pris-, lager-, ordre- og abonnementsdata skal komme fra Shopify eller supportverktøy, ikke fra LLM-kunnskap.

6. QA og rollout

Start med FAQ-only modus, testbenk, intern pilot, begrenset live, logging og review før full synlighet.

Dataflyt

Hva agenten skal vite, og hvor det skal komme fra

OpenAI Responses API for ny agent

Bruk Responses som primær agentflate. Assistants API skal ikke brukes for ny produksjonsbygging.

File Search Godkjente dokumenter i vector store

FAQ, kjøpsvilkår, frakt, retur, produktdata og COA kan søkes semantisk når filene er godkjent og metadata-merket.

Shopify Live data via egne verktøy

Pris, lager, ordrestatus, abonnement og refund må hentes fra Shopify/appdata via sikre server-side verktøy.

Claims Helse og medisiner skal ikke besvares

Sykdom, behandling, graviditet, medisiner, bivirkninger, biomarkører og `antioksidant`-påstander må håndteres av guardrail/handoff.

Personvern Chatlogging er egen policyflate

Personvernerklæring, cookie/CMP, databehandlerforhold, sletting og minimumslogging må avklares før live agent.

Support Ukjent svar blir oppgave

Spørsmål uten sikkert grunnlag skal bli supportticket eller FAQ-backlog, ikke et selvsikkert generert svar.

KB-kontrakt

Metadata som må følge hver kilde

{
  "doc_type": "faq | product | coa | shipping | returns | legal | payment | privacy",
  "approval_status": "draft | approved | expired",
  "language": "nb",
  "product_id": "shopify_product_id_or_pending",
  "batch_id": "batch_id_if_relevant",
  "claim_review": "approved | restricted | blocked",
  "last_reviewed": "YYYY-MM-DD",
  "owner": "role_or_person"
}

Produksjonsagenten skal bare bruke `approval_status=approved`. Draft-kilder kan være med i testbenken, men må blokkeres i live-modus.

Rollout

Lanser agenten i fire kontrollerte trinn

1. FAQ-only

Statisk FAQ/search og foreslåtte spørsmål. Ingen fritekst-LLM mot kunder.

2. Intern agent

Teamet tester fritekst mot godkjente kilder, testbench og røde claims. Logg feil og manglende svar.

3. Begrenset live

Agenten svarer på bruk, frakt, retur og betaling. Medisinsk/claim-sensitivt språk eskaleres automatisk.

4. Utvidet support

Ordrestatus, retur og abonnement kan åpnes først etter sikker autentisering og Shopify-verktøy er testet.

Go/no-go

Ikke aktiver live-agent før alle disse er sanne

Kilder

  • FAQ og produktcopy er claim-reviewet.
  • Produktdata, størrelser, priser og COA er ekte.
  • Frakt, retur, betaling og subscription-vilkår er godkjent.
  • Alle dokumenter har metadata og eier.

Teknikk

  • Server endpoint holder API-nøkler skjult.
  • Rate limiting og abuse-beskyttelse er aktivert.
  • Shopify-verktøy returnerer bare nødvendige data.
  • Feil, timeouts og fallback er testet.

Personvern

  • Chat/AI er omtalt i personvern og supportvilkår.
  • Cookie/CMP og Shopify Customer Privacy API er avklart.
  • Helseopplysninger logges ikke som supportanalyse.
  • Sletterutine og dataeier er definert.

QA

  • Testbenken passerer standardpakken.
  • Medisinsk handoff er testet med norske formuleringer.
  • Ukjent-spørsmål blir supportoppgave.
  • Manuell support kan overta samtalen.
