Preview v1 status

Previewen kan deles. Produksjon er ikke låst.

Dette er en kort beslutningsside for hvor prosjektet står nå: previewpakken er klar som review- og beslutningsgrunnlag, men live Shopify, checkout, frakt, claims og produktdata krever fortsatt kundeinput, kontoer og godkjenninger.

Beslutning

Bruk pakken til kunde-/partnerreview nå. Ikke presenter den som ferdig nettbutikk.

Riktig bruk er å sende hub-lenken, be mottaker starte med reviewguiden, og være tydelig på at produktdata, analysebevis, betaling, frakt, juridikk og live drift fortsatt må fylles inn før Shopify-lansering.

Statuskort

Tre ting som må holdes adskilt

01 Klar for previewdeling

Hub, nettside, produktside, handoff, kravdekning, QA, deploypakke og produksjonsgater er reviewbare.

02 Ikke produksjonsklar

Mangler reelle produktdata, COA, juridisk signoff, Shopify, Vipps/MobilePay, Posten/Bring og testordre.

03 Neste milepæl

Samle produksjonsdata og kontoavklaringer, deretter bygge Shopify med faktiske produkter og checkout.

Riktig inngang

Netlify Drop-roten åpner preview hub. Storefront ligger inne i hubben som `website.html`.

Kildebasert copy

Forsiden bruker levert hero og lede. Intern salgs-/IA-strategi er fjernet fra kundens førsteinntrykk.

Ærlige mangler

Produkt 2, størrelser, priser og COA vises ikke som ferdige data.

Produksjonsgater

Checkout, SEO/AEO, juridisk, accessibility, performance, support, video og opplæring har egne reviewflater.

QA

Customer-copy validator, link audit og focused browser-QA er grønne etter siste pass.

Delingspakke

Lean ZIP er rebuildet uten QA-mappe og macOS-metadata.

Ikke kall dette ferdig

Produksjonsklar betyr noe mer konkret

Kundeinput

  • Produkt 1 og produkt 2 med finalt navn, kategori, størrelse, pris, SKU, vekt og lager.
  • COA/analysebevis med batch, lab, metode, dato, verdier og PDF.
  • Final logo, packshots, produsentvideo, poster, captions, transkript og rettigheter.
  • Godkjente produktpåstander, FAQ-svar, kjøpsvilkår, privacy, cookies, retur og sellerdata.

Kontoer og testing

  • Shopify dev store, domene, SSL og theme upload.
  • Shopify Payments eller valgt kortleverandør.
  • Vipps/MobilePay merchant og faktisk checkout-test.
  • Posten/Bring-oppsett med fraktvalg, hentested, labels, tracking og retur.
  • Testordre, refund, fulfillment, supportagent, Search Console, Merchant Center og Core Web Vitals.

Neste handling

Send previewen, men be om produksjonsdata samtidig.

Praktisk neste steg er å bruke `client-handoff.html` og Netlify Drop-hubben for review, samtidig som kunden fyller `production-intake.html` og beslutningsloggen. Når data og kontoer finnes, kan Shopify-bygg og live checkout testes.
