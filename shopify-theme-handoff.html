Shopify theme handoff

Bruk theme ZIP-en som draft. Ikke publiser den som live butikk ennå.

Denne siden forklarer hvordan `olivex-shopify-theme.zip` skal brukes videre: last opp som upublisert Shopify-tema, koble til ekte produkter og metafields, kjør Theme Check og test checkout/frakt før live.

Beslutning

Theme ZIP-en er et produksjonsutgangspunkt, ikke en ferdig Shopify-butikk.

Den riktige bruken er å laste temaet inn i en Shopify dev store som draft, koble det til faktiske produkter, varianter, media, metafields, betalingsapper og fraktregler, og først publisere etter full testordre og signoff.

Hva ZIP-en inneholder

Dette følger med i scaffolden

OliveX hero

Hero-seksjon med kildebasert heading og bundled hvit transparent OliveX-logo som fallback.

Produktshowcase

Liquid-seksjon for produktkort, variantpris og to-produktstruktur uten hardkodede preview-priser.

Produktside

Buybox, variantvelger, selling-plan readiness, dynamisk pris og mobile sticky add-to-cart.

Metafield-mønster

`lab`-metafields for batch, opprinnelse, lab, metode, sertifikat, dagsmengde og advarsler.

Quality passport

Analysebevis-modul som skal fylles fra produkt/metafields, ikke statisk previewtekst.

Privacy scaffold

Preview-kontroller for samtykke som må kobles til valgt Shopify-kompatibel CMP før produksjon.

Upload som draft

Anbefalt Shopify-admin flyt

01 Opprett dev store

Bruk en butikk der vi kan teste produkter, betaling, frakt og tema uten å påvirke live domene.

02 Last opp ZIP

I Shopify admin: Online Store, Themes, Add theme, Upload zip file. Legg det inn som upublisert tema.

03 Preview før publisering

Åpne theme preview og sjekk hero, menyer, produkter, cart, mobile sticky buybox og policy-lenker.

04 Koble data

Legg inn produkter, varianter, bilder, `lab`-metafields, COA-linker, seller data og policy-innhold.

05 Installer apper

Konfigurer Vipps/MobilePay, Posten/Bring, CMP, analytics og eventuell subscription-app i dev/testmodus.

06 Publiser først etter QA

Ikke publiser temaet før checkout, frakt, claims, schema, accessibility, performance og opplæring er grønt.

cd /Users/andreas/Documents/Prosjekter/2026-Ideas/nordic-longevity-lab/shopify-theme
shopify theme check
shopify theme dev --store <shopify-store.myshopify.com>
shopify theme push --unpublished --store <shopify-store.myshopify.com>

Bruk CLI-flyten når temaet skal itereres mot en konkret Shopify-butikk. Admin upload er nok for første review, men Theme Check og `theme dev` bør brukes før temaet blir produksjonsgrunnlag.

Konfigurasjon

Dette må settes i Shopify før temaet kan vurderes realistisk

Butikkdata

  • Brandnavn, final logo og favicon.
  • Hovedmeny, footermeny, kontaktinfo og selskapsdata.
  • Kjøpsvilkår, personvern, cookies, frakt, retur og angrerett.
  • Domene, SSL, Search Console, sitemap og noindex/indexing beslutning.

Produktdata

  • To produkter med to varianter per produkt.
  • Finale priser, SKU, lager, vekt, fraktdimensjoner og eventuell GTIN.
  • Produktbilder, packshots, produsentvideo, poster, captions og transcript.
  • `lab`-metafields, COA PDF, batch, metode, lab, dato og godkjente claims.

Acceptance test

Minimum før publisering

Theme upload

ZIP laster opp uten manglende layout, schema-feil eller brutte assets.

Theme Check

`shopify theme check` er kjørt og funn er enten rettet eller eksplisitt akseptert.

Produktdata

Ingen `Pris kommer`, placeholder-produkt, tom COA eller uavklart størrelse er synlig i live theme.

Checkout

Testordre med kort, Vipps/MobilePay, capture, refund og kansellering er dokumentert.

Frakt

Posten/Bring viser forventede valg, hentested, rates, etikett, tracking og returflyt.

Mobile

Ingen horisontal overflow, clipped headings, ødelagte bilder eller for små touch targets.

Schema

Product/ProductGroup/Offer, shipping, return policy og VideoObject matcher synlig innhold.

Performance

LCP, INP, CLS og tredjepartsscripts er testet etter apper, video, CMP og supportwidget.
