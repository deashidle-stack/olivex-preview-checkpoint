Produksjonsregel

Mål på ekte mobilopplevelse, ikke bare pen desktop-preview.

Den statiske previewen er lett. Produksjonsrisikoen kommer når Shopify-theme, apper, video, pixels, cookie/CMP, supportwidget og checkout kobles inn. Derfor må ytelse være en lanseringsgate.

Core Web Vitals

Mål som må holdes i produksjon

2.5s LCP

Largest Contentful Paint bør være under 2,5 sekunder for minst 75 % av sidevisningene.

200ms INP

Interaction to Next Paint bør være under 200 ms, også etter app- og checkout-scripts.

0.1 CLS

Cumulative Layout Shift bør være under 0,1. Media, sticky cart, cookiebanner og chat må ikke flytte layout.

Nåværende statisk preview

Utgangspunktet er lett nok til review

Filstørrelsene under er lokale kildefiler, ikke komprimert Shopify/CDN-overføring.

HTML

Forside ca. 28 KB. Produktside ca. 16 KB.

CSS / JS

`styles.css` ca. 40 KB og `app.js` ca. 40 KB. Ingen framework-bundle i previewen.

Hero

Hero JPG ca. 296 KB, preloaded og brukt med eksplisitt bredde/høyde.

Logo

Synlig OliveX-wordmark ca. 40 KB. SVG/optimal eksport bør erstatte raster-logo i produksjon.

Nedover siden

Produkt-placeholder ca. 428 KB og produsentposter ca. 600 KB ligger lenger ned i opplevelsen.

Tredjeparter

Previewen laster ingen analytics, ads, chat, remote fonts eller eksterne scripts.

Performance budget

Anbefalt budsjett for Shopify-produksjon

Critical path

  • Hero/LCP-bilde under ca. 350 KB etter optimalisering.
  • CSS som blokkerer render holdes under ca. 100 KB.
  • Initial JS under ca. 150 KB gzip der det er realistisk.
  • Ingen tredjepartsscript i critical path uten klar verdi.

Media

  • Bruk moderne formater der Shopify/CDN støtter det.
  • Sett width/height eller aspect-ratio på alle media.
  • Lazy-load below-the-fold bilder.
  • Video bruker poster og `preload="metadata"`.

Apps

  • Mål kostnaden av CMP, chat, reviews, subscriptions og shipping app.
  • Fjern app-embed scripts som ikke brukes.
  • Last supportagent først etter samtykke/brukerintensjon hvis mulig.
  • Hold checkout-risiko separat fra storefront-måling.

Stabilitet

  • Sticky kjøp, cookiebanner og chat må ikke dekke fokus eller skyve innhold.
  • Produktkort, størrelsestabeller og badges får stabile dimensjoner.
  • Fontstrategi må unngå store layout shifts.
  • Variantbytte skal ikke endre layoutbredde.

Produksjonsrisiko

Det som mest sannsynlig ødelegger hastigheten

Må måles Shopify-apper

CMP, review-app, subscriptions, shipping, chat og tracking kan legge inn scripts på alle sider.

Må måles Produsentvideo

Video skal ikke autoloade tung fil. Bruk poster, metadata-preload og egen QA på mobilnett.

No-go Uoptimaliserte packshots

Store PNG-er eller originalfoto direkte i theme kan ødelegge LCP og mobilopplevelse.

Må måles Fonts

Eksterne fontkall eller sen fontlasting kan gi treg førstevisning og layout shift.

Må måles Supportagent

Live widget bør lazy-loades og ikke blokkere kjøpsreise, consent eller checkout.

No-go Tracking før consent

Analytics/ads skal ikke lastes før samtykke. Dette er både privacy- og performance-risiko.

Testmatrise

Må kjøres etter Shopify-oppsett

Lighthouse mobile

Forside, produktside, cart state og policy/support-sider etter appene er installert.

Real device

iPhone/Android på mobilnett: hero, variantvalg, sticky purchase, cookiebanner og FAQ/support.

Resource inventory

Liste alle scripts, CSS, bilder, fonts og tredjepartsdomener før noindex fjernes.

Checkout path

Mål handoff til Shopify checkout, Vipps/kort og Posten/Bring uten uventet venting eller layout shift.

Consent states

Test avvis, godta og preferanser. Analytics/ads må kun lastes etter riktig consent.

RUM etter launch

Følg Core Web Vitals i Search Console/CrUX eller valgt RUM-verktøy etter trafikk kommer inn.

Go/no-go

Ikke lanser før dette er dokumentert

Core Web Vitals

LCP, INP og CLS er testet på mobil etter theme, media, app embeds, CMP og support er aktivert.

Mediebudsjett

Hero, packshots, produsentposter og video er optimalisert og har stabile dimensjoner.

Tredjeparter

Alle tredjepartsressurser har eier, formål, consent-status og målt performance-kostnad.

Shopify

Ubrukte app embeds, theme sections og scripts er fjernet før live.

Checkout

Kjøpsflyt, betaling og frakt er testet for hastighet og stabilitet på mobil.

Monitoring

Search Console/Core Web Vitals eller RUM følges etter lansering og første trafikk.
