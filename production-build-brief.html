Produksjonsbrief

Fra preview til ekte Shopify-butikk.

Dette er arbeidsbriefen for å gjøre OliveX-opplevelsen produksjonsklar uten å miste kildedisiplinen, kjøpsflyten eller kravene til trygg handel.

Anbefalt retning

Bygg produksjon som en datastyrt Shopify-butikk, ikke som en statisk kopi av previewen.

Previewen viser opplevelsen. Produksjonen må bygges rundt Shopify-produkter, varianter, metafields, betalingsapper, fraktregler, juridisk godkjent copy og en kontrollert support-kunnskapsbase.

Arkitektur

Produksjonsstack

Shopify theme

Bruk den lokale scaffolden som start. Konverter previewseksjoner til Liquid-seksjoner som henter produkt, variant, COA og mediedata fra Shopify.

Produkter og metafields

Alle størrelser, priser, batchverdier, analysebevis og advarsler skal komme fra produkter/metafields, ikke hardkodes i theme.

Betaling

Kortbetaling settes opp via Shopify Payments eller valgt leverandør. Vipps/MobilePay legges til som faktisk checkout-rute og testordres før live.

Frakt

Posten/Bring må konfigureres med reelle tjenester, hentested, priser, etikett, tracking og returregler i Shopify.

SEO og merchant data

ProductGroup, Product, Offer, shipping og return-policy schema publiseres først når synlige produktdata og butikkregler er endelige.

Supportagent

Agenten bør være server-side, kildebundet og avgrenset. Medisinske spørsmål skal rutes til lege/farmasøyt eller kundeservice, ikke besvares fritt.

Juridisk og policy

Kjøpsvilkår, personvern, cookies, frakt/retur, selskap og produktmerking må være synlige før noindex fjernes.

Performance

Core Web Vitals må måles etter theme, Shopify-apper, video, consent, supportwidget og tracking er aktivert.

Opplæring og drift

Kunden må kunne produkter, metafields/COA, ordre, refund, frakt, support-KB og rapporter før overtakelse.

Dataflyt

Hva som må bli dynamisk

Produktdata

  • Produktnavn, produkt 2-definisjon og kategori.
  • To varianter per produkt med ml, SKU, pris, lager, vekt og eventuell GTIN.
  • Bilder, video, posterbilde, emballasje og endelige logoeksporter.
  • Produsentvideo med norsk teksting, transkript, rettigheter og godkjente kildefakta.
  • Ingredienser, anbefalt daglig mengde, porsjoner, oppbevaring og advarsler.

Dokumentasjon

  • COA/analysebevis med batch-ID, lab, metode, dato og PDF.
  • Godkjente påstander eller trygg produksjonscopy.
  • Frakt-, retur-, personvern-, cookie- og kjøpsvilkår.
  • Juridisk selger/importør, org.nr., adresse, kontakt og supportansvarlig.
  • FAQ-svar som er claim-reviewet og merket med godkjent status.

Faseplan

Anbefalt gjennomføring

01 Datainntak

Lås produktdata, COA, visuelle assets, claims, vilkår og kontoansvar.

02 Shopify setup

Opprett dev store, importer theme scaffold, legg inn produkter, metafields og media.

03 Checkout

Aktiver kort, Vipps/MobilePay og Posten/Bring. Test betaling, capture, refund, frakt og retur.

04 Content lock

Bytt previewcopy til juridisk godkjent live-copy og legg inn ekte analysebevis/video.

05 SEO og agent

Legg inn schema, Merchant Center-data, Search Console og supportagent med godkjent kunnskapsbase.

06 Launch QA

Kjør mobil, desktop, WCAG, checkout, tracking, speed, schema, support og testordre før noindex fjernes.

Go/no-go

Dette må være grønt før lansering

Claims

Ingen sykdoms-, medisin- eller biomarkørpåstander uten juridisk godkjenning og dokumentert grunnlag.

Kjøp

Begge produkter har ekte varianter, priser, lagerstatus og vellykket testordre med kort og Vipps/MobilePay.

Frakt

Posten/Bring viser riktige valg, priser, hentested, etikett, tracking og returflyt.

SEO

Synlig produktdata matcher schema, Merchant Center, canonical, sitemap og Search Console.

Personvern

Consent-banner, privacy policy, cookieoversikt og GA4/ads behavior er godkjent og testet.

Performance

LCP, INP, CLS, medievekt, tredjepartsscripts og Shopify-app embeds er testet på mobil før live.

Support

FAQ/supportagent svarer bare fra godkjent kunnskap og eskalerer medisinske spørsmål.

Opplæring

Kunden har gjennomført praktiske øvelser for produktendring, testordre, refund, tracking og support-handoff.

Kilder sjekket 7. juni 2026

Produksjonsantakelser som må følges

Shopify Betaling

Shopify betalingsmetoder administreres i Shopify admin. Abonnement krever særskilt gateway-/subscription-oppsett.

Vipps Checkout

Vipps/MobilePay-pluginen lar kunder velge Vipps/MobilePay i checkout, men Express er ikke støttet i Shopify per dokumentasjonen.

Bring Frakt

Posten/Bring må testes med live tjenester, pickup point, labels og tracking før det loves som produksjonsklar flyt.

Google Variantschema

Google anbefaler ProductGroup/Product for varianter, validert med Rich Results Test og URL Inspection før indeksering.

Consent Privacy API

Shopify Customer Privacy API brukes for consent på Shopify-managed surfaces. Ikke manipuler Shopify-cookies direkte.

AI support File search

Produksjonsagenten bør bruke godkjent kunnskapsbase og retrieval, med logging og handoff for medisinske tema.

Neste praktiske steg

Be kunden fylle produksjonsdata før Shopify bygges.

Før vi låser designet i Shopify bør kunden levere produkt 1 og produkt 2 med størrelser, priser, COA, bilder, video, vilkår, fraktvalg og godkjente påstander. Uten dette må butikken fortsatt være i previewmodus.
