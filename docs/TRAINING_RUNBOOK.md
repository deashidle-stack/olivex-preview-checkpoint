# Training Runbook

Use this as the agenda for the client training session.

Browser-native readiness page:

- `training-ops-readiness.html`

Expanded current runbook:

- `docs/TRAINING_OPS_READINESS_2026-06-07.md`

## Session 1: Store Basics

- Log in to Shopify Admin
- Find products
- Edit product text
- Edit product images
- Edit variant prices
- Edit inventory
- Edit SEO title and meta description

## Session 2: Quality Passport

- Open product metafields
- Update batch ID
- Update origin
- Update lab name
- Update test method
- Add certificate link
- Update serving size and daily dose
- Add warnings

Rule: do not add or change health claims without legal review.

## Session 3: Orders

- Find new orders
- Understand payment status
- Capture payment if configured manually
- Print or generate shipping label
- Add tracking
- Mark fulfilled
- Handle refunds

## Session 4: Content

- Edit homepage sections
- Edit product showcase
- Edit Quality Passport section fallback text
- Edit FAQ/legal pages
- Create Lab Notes article
- Add internal links from article to product

## Session 5: Integrations

- Shopify Payments overview
- Vipps/MobilePay plugin overview
- Posten Bring Checkout overview
- Cookie consent tool overview
- GA4/Search Console overview

## Handoff Checklist

- Client owns Shopify account
- Client owns domain
- Client owns payment provider accounts
- Client owns Vipps merchant agreement
- Client owns Posten/Bring agreement
- Client has admin access
- Client has written runbook
- Client knows who approves legal claims
- Client knows who owns product data, COA/metafields, support KB, shipping, payments, analytics and technical escalation
- Client has practiced test order, refund, tracking and product update before launch
